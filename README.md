# 🛡️ SentinelCore Compliance & Security Gateway

SentinelCore is an open-source, highly portable, **self-hostable developer security proxy, compliance gateway, and real-time middleware auditor**. 

It is designed to solve the SaaS trust hurdle (typical of walled gardens like Cursor, Lovable, Replit, or Vercel) by giving developers complete ownership of their security pipeline with **Bring Your Own Key (BYOK)** support. 

---

## ✨ Features

- **Multi-Tier Threat Defense**:
  - **Tier 1 (Local Deterministic Edge)**: Instant regex, keyword, and pattern matching running physically offline under 10ms.
  - **Tier 2 (High-Fidelity Semantic Scan)**: Cognitive intent mapping, deceptive neutralized framing, and vulnerability vector detection.
  - **Tier 3 (Interactive Adversarial Debate)**: Multi-agent debate (Prosecutor, Defense, Judge) to evaluate borderline prompts prior to downstream generation.
- **OpenAI-Compatible Proxy Engine**: Gating endpoint located at `/v1/chat/completions` for direct integration into any developer workspace (Cursor, VS Code, Replit, etc.).
- **Express Gating Middleware**: Mountable middleware to protect your company's proprietary Node.js APIs and AI-driven apps.
- **Real-Time Interactive Dashboard**: Built-in React UI to review scans, audit latency, calculate exact costs, track compliance warnings (HIPAA, GDPR, EU AI Act), and monitor developer traffic.

## 🚀 Local Quickstart (Run On Your Own Machine)

SentinelCore is **fully self-contained** and designed to be run locally in any Node.js environment.

### 1. Install Dependencies
```bash
npm install
```

### 2. Configure Environment Variables
Create a `.env` file in the root directory:
```env
GEMINI_API_KEY=your_gemini_api_key_here
```

### 3. Start Development Server
```bash
npm run dev
```
This boots the Express proxy backend and the React hot-reload dashboard on **`http://localhost:3000`**.

## 🔌 IDE Integration (Cursor, VS Code, Lovable, Replit)

Once SentimentCore is running locally on your laptop or inside your private VPC, you can route all AI workspace traffic through it to enforce compliance and data leakage rules automatically.

### Configuration
Update your IDE's OpenAI-compatible proxy settings:
- **API Base URL**: `http://localhost:3000/v1`
- **Model**: `gemini-3.5-flash` (or your preferred compatible model)
- **API Key**: Any placeholder (SentinelCore uses your private `GEMINI_API_KEY` defined in step 2 securely, or relays keys based on your BYOK preference).

### Real-Time Interception Output
If a developer prompts their coding assistant with a rule violation (e.g., trying to write raw SSNs, violating HIPAA guidelines, or performing systemic jailbreak overrides):
1. **The request is blocked instantly prior to downstream LLM transmission** (saving cost and avoiding leakage).
2. The IDE Assistant receives a clean structural notification:
   ```text
   ⚠️ [SentinelCore Gating Blocked]
   Your prompt triggered a security or compliance rule in your developer workspace.
   🔒 REASON: Semantic PII leakage detected. Ensure HIPAA boundaries are respected.
   📉 RISK SCORE: 8/10
   ```
3. The event, latency metrics, and compliance framework status immediately populate your local dashboard at `http://localhost:3000`.

## 🌐 Express Middleware Integration

You can integrate the core SentinelCore gate directly into your own Node.js backend. Let's secure your prompt-based endpoints in three lines of code:

```typescript
import express from "express";
import { sentinelCore } from "./lib/sentinel-core";

const app = express();
app.use(express.json());

// Register unified sentinelCore middleware on your LLM routes
app.post(
  "/api/generate",
  sentinelCore.express({
    promptField: "prompt",            // Field in req.body to scan
    appId: "customer_support_app",   // Application audit profile ID
    failSecure: "BLOCK",             // Fail secure policy (reject with 403 on risk)
    scanTools: true,                 // Scan tool calls alongside text
  }),
  async (req, res) => {
    // If execution reaches here, the prompt is validated as 100% compliant and safe!
    const auditTelemetry = req.sentinelVerdict; // Contains full score details
    res.json({ text: "Safe response generated by your downstream model." });
  }
);
```

## 🛠️ Production Build & Container Deployment

To compile SentinelCore into a fast CJS bundle optimized for Docker, AWS ECS, GCP Cloud Run, or serverless execution:

```bash
# 1. Build and compile assets
npm run build

# 2. Run the production-optimized server bundle
npm start
```
The build system uses `esbuild` to package the server runtime into a single standalone `dist/server.cjs` file, allowing seamless, highly performant cold starts in zero-install container environments.
